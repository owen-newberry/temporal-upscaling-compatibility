────────────────────────────────────────────────────────────────────────────────
PROJECT GOAL
────────────────────────────────────────────────────────────────────────────────
Build 4 small UE5 C++ demo systems that demonstrate simulation correctness
vs incorrect design patterns. Each demo lives in its own level inside the
single "demos" UE project:

  1. Motion Authority System          ← CURRENT FOCUS (in progress)
  2. Fixed Timestep vs Variable Timestep
  3. Time-based vs Frame-based Motion
  4. Workload Budgeting (task queue per frame)

The demos are intended as a capstone/research project exploring how different
architectural choices (authority, timestep, motion model) affect simulation
fidelity and compatibility with temporal upscalers (TSR, TAA, DLSS, etc.).

WHAT IS IMPLEMENTED (Demo 1 — Motion Authority)
────────────────────────────────────────────────────────────────────────────────
Two C++ Actor classes have been written and successfully compiled:

── AMoverActor (MoverActor.h / MoverActor.cpp) ──────────────────────────────
  Purpose: A client actor that represents one moving cube.

  Key properties (all editable in Details panel):
    UStaticMeshComponent* Mesh          — auto-assigned /Engine/BasicShapes/Cube
    AMotionAuthorityActor* AuthorityManager  — assign in editor; null = direct mode
    FVector MoveVelocity = (100, 0, 0)  — cm/s along X
    FVector MoveForce    = (0, 0, 0)    — additional force

  Tick behaviour:
    IF AuthorityManager is assigned AND bAuthorityMode == true:
      → Submits (Velocity + Force) to the manager. Does NOT touch its own transform.
    ELSE (direct mode, AuthorityManager is null):
      → Calls AddActorWorldOffset((MoveVelocity + MoveForce) * DeltaTime)
      → Logs position + delta to Output Log (Error level, always visible)
      → Displays live position on screen via GEngine->AddOnScreenDebugMessage

  Debug output (currently active — ERROR level so it always shows):
    "[MoverActor] BeginPlay fired: <name>"          — on spawn
    "[MoverActor] TICK <name> | pos <xyz> | delta X cm"  — every frame

── AMotionAuthorityActor (MotionAuthorityActor.h / .cpp) ────────────────────
  Purpose: Central manager that collects movement inputs and applies them.

  Key properties:
    bool bAuthorityMode = true   — toggle in editor (true = authority, false = direct)

  API:
    void SubmitInput(AActor* Actor, FVector Velocity, FVector Force)
      — Called by client actors each frame in authority mode.

  Tick behaviour:
    Iterates PendingInputs[], calls SetActorLocation on each actor,
    logs position delta, detects jitter (frame-to-frame delta change > 50 cm),
    then clears the list.

  Jitter detection:
    Compares FrameDelta to previous frame's delta per actor.
    Flags if difference > 50 cm with a Warning log.

── FMotionInput (struct in MotionAuthorityActor.h) ──────────────────────────
  Holds: AActor* Actor, FVector Velocity, FVector Force

HOW TO BUILD ON WINDOWS
────────────────────────────────────────────────────────────────────────────────
1. Install Unreal Engine 5.7 via the Epic Games Launcher
2. Clone the repo:
     git clone https://github.com/owen-newberry/temporal-upscaling-compatibility.git
3. Right-click demos.uproject → "Generate Visual Studio project files"
4. Open demos.sln in Visual Studio
5. Set config to Development Editor / Win64 → Ctrl+Shift+B
6. Double-click demos.uproject to open in the UE editor
7. Open level L_MotionAuthority, hit Play — cubes should move along X axis
